# DouDou 微课程一号「形状小画家」设计

日期：2026-07-22
状态：已与产品负责人逐节确认，待实施（课程配置功能属后续里程碑，本文档同时给出其数据模型需求）

## 1. 背景与定位

DouDou 是面向 3-7 岁中文儿童的护眼学习伙伴（手机听说 + 服务器大脑 + reMarkable Paper Pro 纸面）。
服务器一期（见 `2026-07-22-doudou-server-phase1-design.md`）提供两条互不联动的通路：

- **手机语音**：按住说话 → STT → 回复 → TTS 播放；
- **平板手写问答**：孩子写/画 → 发送页面 → AI 手写短句回复。

本文档设计 DouDou 的第一套微课程，并定义后台「课程配置」功能所需的数据模型。

**课程定位**：

- 课程名：**形状小画家**（curriculum slug：`shapes-01`）
- 对象：**3-4 岁涂鸦期儿童**，全程需家长陪同
- 规模：**8 课**，每课约 10 分钟；建议每周 2-3 次，3-4 周完成一轮；同一课与整套课程均可重复上（3-4 岁重复即巩固）
- 主线：沿涂鸦发展梯度——控笔线条 → 基本形状 → 组合成物 → 自由创作
- 设备形态：**语音主线 + 平板提交点**。引导、对话、鼓励全部走手机语音；每课设一个「把画给 DouDou 看」时刻，孩子在平板上触发发送，DouDou 手写夸奖 + 小符号，家长念给孩子听。两条通路一期不联动，靠家长在明确的串场点衔接。

**能力标注原则**：全部 8 课的核心活动只依赖现有能力（语音对话 + 平板自由涂鸦 + 提交后手写短句回复）即可运行。「简笔画示范」「描形练习格」「形状卡片」等依赖二期结构化渲染的活动，一律作为**二期增强**逐课标注，不阻塞首套课上线。

## 2. 范围边界

本课程设计**不做**：

- 环节级服务端状态机（软脚本，环节推进由 AI 依对话掌握，见 §6）
- 达标判卷（宽松参与制：动手画了、有互动即完成）
- 平板通路的课程感知（一期平板保持自由问答；提交轮的夸奖规则写进 profile 人设，见 §6.5）
- 跨设备联动（沿服务器一期边界）
- 颜色教学目标（Paper Pro 电子纸色彩偏淡；颜色词只在语音里自然带过）
- 数数教学目标（「数一数画了几个」作为顺带互动，不设目标、不判定）
- 周报（课后反馈只做每课小结卡）
- 多孩子档案（一期单孩子；孩子称呼建议家长直接写入 profile 人设文本，课程表不单独存孩子档案）

## 3. 课程总体设计：8 课列表

每课 = 一个「形状朋友」+ 生活联想 + 涂鸦任务。⟲ 为天然综合复习课。

| # | 课名 | 教学目标 | 涂鸦任务（生活联想） |
|---|------|----------|----------------------|
| 1 | 认识 DouDou·想画就画 | 破冰：敢拿笔、留下痕迹、认识 DouDou、跑通全流程 | 自由涂鸦（笔宝宝想在纸上跳舞） |
| 2 | 长长的线 | 有控制地画线（横/竖/波浪任一即可） | 下雨啦、煮面条、弯弯的小路 |
| 3 | 圆圆的朋友 | 画封闭的圆（歪扭都算） | 吹泡泡、大太阳、气球 |
| 4 | 圆和线做朋友 ⟲ | 组合①：圆+线；复习第 2、3 课 | 太阳发光芒、气球拉线、棒棒糖 |
| 5 | 方方的朋友 | 画方形（「转弯回家」的四边意识） | 小饼干、小窗户、积木 |
| 6 | 尖尖的三角 | 画三角形（「上坡—下坡—回家」） | 小山、小旗子、冰淇淋筒 |
| 7 | 形状变变变 ⟲ | 组合②：用 2-3 种形状拼成一个东西，并说出用了什么形状 | 给小兔子盖房子（方+三角）/ 做冰淇淋（圆+三角） |
| 8 | 我的小画展 🏅 | 自由创作一幅画并讲述（语言表达）；奖励仪式 | 画最喜欢的画讲给 DouDou 听；DouDou 手写颁「小画家奖」 |

每课开头固定有 1-2 分钟滚动复习上一课形状（见 §4 环节①），第 4、7、8 课再叠加综合复习。

## 4. 单课五环节模板

每课同构，约 10 分钟，只替换形状与联想物。以第 3 课「圆圆的朋友」为例：

| 环节 | 时长 | 通路 | 内容 |
|------|------|------|------|
| ① 问好与复习 | 1-2' | 语音 | 固定开场仪式 + 叫孩子名字 + 复习上一课：「还记得长长的线吗？用小手在空中画一条给我看～」（空中比划，家长口头反馈）。第 1 课无复习，改为 DouDou 自我介绍。 |
| ② 认识新形状 | 2' | 语音 | 引入「形状朋友」+ 生活联想问答：「圆圆的，像什么呀？」孩子说什么都接住并肯定，再补 1-2 个联想。让孩子用手指在空中/桌上描一描。 |
| ③ 动手画 | 4-5' | 平板 + 语音陪伴 | 布置任务，句式只含一个动词一个名词：「我们来吹泡泡吧！」孩子在平板上画；期间家长可按住说话让孩子汇报（「我画了三个泡泡！」），DouDou 语音回应鼓励。不教「正确画法」、不纠正形状。 |
| ④ 给 DouDou 看 | 1' | 平板提交点 | 家长帮孩子触发平板发送。此轮 AI 真正「看得见」画——手写回复固定格式：一句具体的夸（点出画面细节）+ 一个手写小符号奖励（⭐/❤/☺）+ 可选一个小问题。家长念给孩子听。 |
| ⑤ 鼓励收尾 | 1' | 语音 | 总结今天的形状朋友 + 仪式感告别（「给自己拍拍手！」）+ 预告下一课 + 输出课程打标（含家长小结卡内容，落库不播报，见 §6.3）。 |

**「看不见」的补偿话术**：语音侧 DouDou 永远不假装看见画，而是问「给我讲讲你画了什么呀」——把通路限制转化为语言表达训练点；真正的「看见」只发生在第④环节。

**弹性规则**（写进每课脚本模板）：

- 孩子跑题 → 跟随半分钟再温柔拉回；
- 不想画 → 改成空中画，或孩子指挥家长画；
- 提前画完 → 追加「再画一个更大的」；
- 烦躁 → 任何环节可直接跳到收尾，参与即完成（status=partial）。

## 5. 8 课详案

每课字段：目标 / 复习 / 联想引导 / 涂鸦任务 / 话术要点 / 提交轮夸点 / 二期增强。

### 第 1 课 认识 DouDou·想画就画

- **目标**：破冰。敢拿笔、在纸上留下任何痕迹、认识 DouDou、家长跑通两台设备的完整流程。
- **复习**：无。改为自我介绍：「我是 DouDou，我最爱看小朋友画画啦。」
- **联想引导**：笔宝宝想在纸上跳舞，它跳什么舞都好看。
- **涂鸦任务**：自由涂鸦，画什么都可以。
- **话术要点**：零要求。孩子画完任何东西都问「哇，给我讲讲这是什么呀」；**不替孩子命名画作**（不说「这是太阳吧」，让孩子自己命名，说什么都对）。
- **提交轮夸点**：夸「痕迹」本身——线条多、下笔大胆、深浅变化。
- **二期增强**：DouDou 简笔画自画像卡（让孩子认识 DouDou 的形象）。

### 第 2 课 长长的线

- **目标**：有控制地画线，横线/竖线/波浪线任一即可。
- **复习**：「上次笔宝宝跳了舞，我们在空中再挥一挥！」
- **联想引导**：下雨（竖线）、煮面条（波浪线）、弯弯的小路（横线）。
- **涂鸦任务**：「下雨啦，画好多好多雨丝！」（孩子若更爱面条/小路，随孩子选）。
- **话术要点**：方向词 +拟声词帮节奏——「从上往下，唰——」；画完可数一数几根（顺带数数，不判定）。
- **提交轮夸点**：点出线的方向、长短、数量：「这么多长长的雨丝，哗啦哗啦下大雨啦！」
- **二期增强**：描线练习格（虚线雨丝描一描）。

### 第 3 课 圆圆的朋友

- **目标**：画封闭的圆，歪歪扭扭都算。
- **复习**：空中画长长的线。
- **联想引导**：泡泡、大太阳、气球。
- **涂鸦任务**：「我们来吹泡泡吧，画大大小小的圆泡泡！」
- **话术要点**：封闭概念说成「让笔绕圈圈，转呀转，让线的头和尾巴牵上手」；先大圆再小圆。
- **提交轮夸点**：圆的数量与大小对比（「这个泡泡好大，旁边还有一个小的！」）；头尾没牵上手**不作为批评点**。
- **二期增强**：描圆练习格。

### 第 4 课 圆和线做朋友（组合①，复习课）

- **目标**：把两种旧元素组合起来（圆+线）。
- **复习**：空中画圆 + 空中画线（两个都来一遍）。
- **联想引导**：太阳发光芒、气球拉绳子、棒棒糖。
- **涂鸦任务**：「给太阳画光芒」或「给气球拴绳子」（选一）。
- **话术要点**：先画圆，再「让线从圆身上长出来」；组合说成「让圆和线做朋友」。
- **提交轮夸点**：点出组合关系：「气球拉着长长的线，要飞起来啦！」
- **二期增强**：DouDou 简笔画示范卡（示范一个发光的太阳）。

### 第 5 课 方方的朋友

- **目标**：画方形，建立「转弯、回家（闭合）」的四边意识，歪扭都算。
- **复习**：空中画圆。
- **联想引导**：小饼干、小窗户、积木。
- **涂鸦任务**：「画一块小饼干」（可以给饼干点上芝麻点点）。
- **话术要点**：口诀化——「画一条线，转个弯，再转个弯，再转个弯，回家啦！」
- **提交轮夸点**：夸转弯和角：「这块饼干转了好几个弯，角角真神气！」
- **二期增强**：描方练习格。

### 第 6 课 尖尖的三角

- **目标**：画三角形。
- **复习**：空中玩「转弯回家」画方游戏。
- **联想引导**：小山、小旗子、冰淇淋筒。
- **涂鸦任务**：「画一座小山」（画得开心可以画一排山）。
- **话术要点**：三步口诀——「上坡——下坡——回家！」；强调「尖尖的头」。
- **提交轮夸点**：夸尖角：「这座山的头好尖，都戳到云朵啦！」
- **二期增强**：描三角练习格。

### 第 7 课 形状变变变（组合②，综合复习课）

- **目标**：用 2-3 种形状组合成一个东西；能说出用了哪些形状。
- **复习**：空中快闪游戏——依次画圆、方、三角。
- **联想引导**：方+三角=小房子；圆+三角=冰淇淋。
- **涂鸦任务**：「给小兔子盖一座房子」或「做一个冰淇淋」（选一）。
- **话术要点**：先问「房子要用哪些形状朋友呀？」让孩子说；然后一步一形状（「先画方方的墙……再给它戴上三角形的屋顶帽子」）。
- **提交轮夸点**：点出用到的形状名：「方方的墙加尖尖的屋顶，小兔子的新家真漂亮！」
- **二期增强**：形状卡片展示（圆/方/三角卡片供拼装联想）。

### 第 8 课 我的小画展（汇报课）

- **目标**：自由创作一幅画并讲述出来（语言表达是本课主目标）；完成课程奖励仪式。
- **复习**：「我们认识了哪些形状朋友呀？」让孩子报形状名（说几个都对，DouDou 补全）。
- **涂鸦任务**：「画一幅你最喜欢的画，画完讲给我听。」
- **话术要点**：讲述引导三问——画的是什么？它在做什么？你最喜欢哪里？每答一问都热情回应。
- **提交轮夸点**：手写「小画家奖」三个大字 + 星星/奖杯符号 + 引用孩子画面元素的一句赞美。家长郑重念出来，建议保留这一页。
- **小结卡特别版**：8 课总回顾——从历史 lesson_runs 的 highlights 汇总孩子这一轮的成长（「从第一课的随手涂鸦，到今天会用三种形状盖房子」）。
- **二期增强**：结构化奖状卡片渲染。

## 6. AI 执行设计

### 6.1 软脚本：课时脚本模板

每课一份结构化脚本文本（`lessons.script_text`）。语音轮组装 system prompt 时追加在「生效 profile 人设 + voice_hint」之后：

```
【今天的课】形状小画家 · 第 3 课「圆圆的朋友」
【上节课回顾】{prev_lesson_recap ← 上一课 lesson_run 的 highlights + parent_tip 注入；首课为空}
【本课目标】敢画封闭的圆；说出 1 个圆的生活联想。参与即成功，不设达标线。
【五环节脚本】
① 问好，叫孩子名字；复习：空中画长长的线
② 认识圆：问「圆圆的像什么」，接住任何答案；补充联想泡泡/太阳；空中描圆
③ 布置任务：画大大小小的圆泡泡；孩子画时语音陪伴鼓励；封闭说成「头尾牵手」
④ 请家长帮孩子把画发给 DouDou 看（平板上发送）
⑤ 夸奖收尾，给自己拍拍手；预告下次「方方的朋友」；输出课程打标
【话术规则】【弹性规则】【判定与打标】（固定规则，见 6.4；打标协议见 6.3）
```

环节推进由 AI 依会话历史自然掌握，**不做服务端环节状态机**。一节课 = 手机页上的一个会话（会话历史由客户端持有，贴合一期架构）。

### 6.2 课程触发形态

沿用一期「全局唯一生效 profile」的先例：

- 后台课程配置设 **全局生效课程**（`curricula.status=active` 互斥）+ **当前课时指针**（`curricula.current_lesson_id`）。
- 家长在手机页点「开始上课」：服务器创建 `lesson_run`（返回 `lesson_run_id`），此后本会话每个 voice-turn 请求携带该 id；服务器组装 system 时注入当前课时脚本。
- 不上课时手机页维持普通语音闲聊（不注入课程脚本）。
- 打标 `status=completed` 后服务器自动把 `current_lesson_id` 推进到下一课；`partial/skipped` 不推进；家长可在后台手动改指针（跳课、重上）。第 8 课完成后 `current_lesson_id` 置空（指针为空且存在末课完成记录即视为本轮完成，status 枚举不另设 completed），后台可一键重开新一轮。

### 6.3 打标协议 ⟦lesson_report⟧

判定与小结卡一次产出。课程收尾轮（环节⑤），AI 在回复文字末尾追加一行标记：

```
⟦lesson_report⟧{"lesson_id":"shapes-01-03","status":"completed",
"highlights":"画了5个大小泡泡，还给泡泡点了眼睛",
"parent_tip":"在家吹真泡泡时，让他用手指跟着泡泡画圈"}
```

（示例为排版换行，实际协议要求标记与 JSON 在同一行。）

- `status`：`completed`（正常走完）| `partial`（中途跳收尾）| `skipped`（几乎没参与）。
- 服务器解析后从展示文本剥离，**绝不进入 TTS 播报**；解析结果写入 `lesson_runs`。
- `highlights + parent_tip` 即家长小结卡内容，也是下一课 `{prev_lesson_recap}` 的注入来源——滚动复习闭环。
- 解析失败兜底：整轮文本原样落库、`lesson_run.status` 记 `abandoned`（含家长中途关页超时未收尾的情况），家长可在后台补记。

### 6.4 鼓励与纠错固定规则（每课共用，写入脚本模板尾部）

- **夸具体行为不夸聪明**：「这个泡泡好大好圆」✓，「你真棒真聪明」✗。
- **不纠错**：画不圆不说「不对」，把重试藏进新任务——「哇，这个泡泡在跳舞！要不要再吹一个慢慢的圆泡泡？」
- **「不会画」→ 拆到最小动作**：「先点一个点，让笔绕着点跑一圈」；允许家长手把手。
- **每轮语音只说 1-2 句、一个指令**，不连环提问。
- **STT 现实规则**：3-4 岁口齿不清、转写常出错——听不懂时不猜硬答，温柔请孩子再说一遍或请家长帮忙转述。

### 6.5 平板提交轮（儿童画作回应规则）

一期平板通路不感知课程。第④环节的手写夸奖依靠把以下规则写进 **3-4 岁 profile 人设文本**（现有能力即可运行）：

> 收到孩子的涂鸦画作时：回复第一句是具体的夸奖，必须点出画面里至少一个真实元素；
> 然后手写画一个小符号奖励（⭐、❤ 或 ☺）；最后可以加一个关于画的小问题。
> 全文不超过 3 行，不评价画得像不像，不提「图片/照片」。

二期增强：门面注入当前课时上下文，夸奖能点出「今天学的形状」。

## 7. 家长角色与课后反馈

**课前**：家长看 `lessons.materials`（课前准备说明：手机开 DouDou、平板进画板、安静角落、约 10 分钟）。

**课中三顶帽子**：

1. **操作员**：按住说话、第④环节帮孩子触发平板发送、念手写回复；
2. **传声筒**：孩子口齿不清时向 DouDou 复述；
3. **同伴**：自己也画一个，画得不好正好示范「不完美也开心」。

**明确约定**（写入课程介绍页）：家长不当老师、不纠正像不像、不代笔（手把手除外）。

**课后小结卡**：每课收尾自动产出（§6.3），家长在后台课程记录页查看：

- 今天：第 3 课 圆圆的朋友 · 已完成
- 亮点：画了 5 个大小泡泡，还给泡泡点了眼睛
- 在家延伸：吹真泡泡时，让他用手指跟着泡泡画圈
- 作品：本课关联的平板提交图（时间窗自动挂靠，见 §8）

## 8. 课程数据模型需求（供后台「课程配置」功能设计引用）

延续一期 SQLite/SQLAlchemy 风格：**新增 3 张表 + turns 表加 1 列**。

### curricula（课程）

| 字段 | 说明 |
|------|------|
| id, slug, title | `shapes-01`「形状小画家」 |
| age_band | "3-4"（与 profiles.age_band 同枚举） |
| description | 课程简介（家长可见） |
| status | draft / active / archived；**active 全局唯一**（沿生效 profile 互斥模式） |
| current_lesson_id | 当前课时指针（可空；空 + 有完成记录 = 本轮完成） |
| created_at, updated_at | |

### lessons（课时）

| 字段 | 说明 |
|------|------|
| id, curriculum_id, seq | 第几课（1-8） |
| slug, title | `shapes-01-03`（课程 slug + 两位课序）「圆圆的朋友」 |
| goal_text | 本课目标 + 判定口径（宽松参与制描述） |
| script_text | 五环节课时脚本正文（§6.1 全文，语音轮直接注入） |
| segments | JSON 结构化环节数组：`[{seq, kind: warmup\|teach\|draw\|submit\|closing, channel: voice\|tablet, goal, script_hints}]`。一期只用于后台课时编辑器渲染与拼装 script_text，**不驱动服务端状态**；二期作为对齐平板 paper_cards 的挂载点。不单设 activities 表——软脚本下活动即环节，避免过度建模。 |
| duration_min | 目标时长（10） |
| materials | 给家长的课前准备说明 |
| enhancements | JSON，二期增强项标注（描形格/示范卡/形状卡片等） |
| updated_at | |

### lesson_runs（课时运行记录）

| 字段 | 说明 |
|------|------|
| id, lesson_id | |
| started_at, ended_at | 手机页「开始上课」创建 / 收尾打标或超时关闭 |
| status | completed / partial / skipped / abandoned（abandoned=未收到打标） |
| highlights, parent_tip | 小结卡内容（来自 ⟦lesson_report⟧） |
| raw_report | JSON，打标原文解析结果（含解析失败时的原始文本） |
| memory_tags | JSON，一期恒空——二期喂 mem0/Memobase 的挂载点 |
| artifact_turn_ids | JSON，关联平板提交轮 turns.id（作品存档） |
| parent_note | 家长补记（可空） |

同一课可多次上 → 多条 run；`{prev_lesson_recap}` 取上一条非 abandoned 的 run。

### turns 表加列

`lesson_run_id`（nullable FK）：语音轮由手机页请求携带自动归档。平板提交轮因两通路不联动无法携带——按**时间窗自动挂靠**：提交时间落在某 run 起止窗内的 tablet 轮自动写入该 run 的 `artifact_turn_ids`，家长后台可解除或手动挂靠。

### 明确不改

profiles、providers、voice_settings 表零改动；课时脚本在 turn 组装时动态拼接，不落 profile。

## 9. 与并行工作的对齐

- **服务器一期实施**（并行窗口）：本设计只依赖一期已定稿能力（run_turn 引擎、profile 组装、手机 voice-turn、turns 落库）。课程功能全部属于后续里程碑，一期不需要为本设计改动任何已定稿接口；唯一的前向兼容请求是 turns 表将来可加 `lesson_run_id` 列（SQLite 加列为轻量迁移）。
- **平板交互设计**（并行会话，文档 `2026-07-22-tablet-interaction-design.md` 截至本文定稿尚未产出）：本课程活动形式只依赖「自由画布 + 手写短句回复」两个现有原语，不与其冲突。该文档产出后，在此补一节 `segments.kind` ↔ paper_cards 类型的映射表，并把各课 `enhancements`（描形格、示范卡、形状卡、奖状卡）改写为其结构化类型。

## 10. 二期挂载点清单

1. `lesson_runs.memory_tags` → 记忆系统（mem0/Memobase）：课程侧产出「孩子会画圆了」「喜欢画泡泡」类标签。
2. `lessons.segments` → paper_cards 结构化渲染（描形格、简笔画示范、形状卡片、奖状卡）。
3. 平板通路课程感知：门面注入当前课时上下文，手写夸奖点出今日形状；平板提交轮可选做形状辅助判定（仅作参考，不改变宽松参与制）。
4. 跨设备联动后：第④环节可由语音侧直接「看见」提交的画，补偿话术随之退役。
5. 后续课程扩展：同数据模型可承载 5-6 岁识字启蒙课（`age_band="5-6"`，segments 增加识字类 kind）。
